========================
CURRENT SYSTEM SUMMARY
========================

Program: Spotify Playback Controller

Current capabilities:
- Authenticates user via Spotify OAuth (Spotipy)
- Accepts a Spotify playable resource as input
- Re-prompts until a valid playable Spotify resource is entered
- Supports playlist, album, artist, track, and episode resources
- Detects current playback state before acting
- For Premium users:
  - resumes current playback if the requested resource already matches
  - continues playback when paused on the requested resource
  - otherwise starts playback from the beginning of the requested resource
  - supports optional stop logic by duration or clock time
- For non-Premium users:
  - does not attempt premium-only playback control
  - prints a clear Premium requirement message
  - shows playlist contents or readable resource metadata instead

Playback-state awareness:
- Checks:
  - is_playing
  - current item URI
  - current context URI
  - progress_ms (when available from the API)
- Uses that information to decide between resume/continue/restart behavior

Non-Premium support:
- Gracefully handles users without Premium
- Uses metadata-reading endpoints even when playback control is unavailable
- For playlists, prints a readable track list
- For albums, prints album metadata and track names
- For artists, prints artist metadata
- For tracks and episodes, prints readable metadata

Structure:
- get_spotify_client()
- get_active_device()
- extract_spotify_resource()
- validate_resource()
- prompt_until_valid_resource()
- get_current_playback()
- determine_premium_status()
- show_resource_contents()
- start_or_resume_playback()
- stop_after_duration()
- stop_at_clock_time()
- main()

Inputs:
- Spotify URL or URI
- stop mode (duration/time/none) for Premium playback path
- timing input when playback control is available

Outputs:
- connection and validation status messages
- Premium requirement message for non-Premium users
- playlist contents / resource metadata in non-Premium mode
- playback control behavior in Premium mode

Key Variables:
- resource_input
- resource_uri
- resource_type
- resource_id
- resource_data
- playback
- is_premium
- device_id
- current_context_uri
- current_item_uri
- progress_ms

Design Principles:
- Always validate input
- Never assume playback state
- Preserve user context when possible
- Separate parsing, validation, playback, and display logic
- Fail gracefully for non-Premium users
